## Naming Convention

Screenshots follow the pattern: `radio_{category}_{details}_{theme}.png`

- **category**: `yes_no`, `climate`, `languages`, `animation`
- **details**: Selection state, language, or animation type
- **theme**: `free_light`, `free_dark`, `dreamer_light`, `dreamer_dark`

## Usage

These screenshots serve as:
- **Visual Regression Tests**: Detect unintended visual changes
- **Design Documentation**: Show component appearance across themes
- **Quality Assurance**: Verify consistent rendering
- **Development Reference**: Visual examples for developers

## Regenerating Screenshots

To regenerate all screenshots:
```bash
./gradlew recordRoborazziDebug
```

To verify screenshots match:
```bash
./gradlew testVisual
```

To record and copy to BEM structure:
```bash
./gradlew testVisualSave
```
